## Process
When you receive a task, follow these steps in order.

**File Usage & Efficiency:** Always use files to type notes, track progress, and keep references to prevent forgetting. Use append, edit, and copy‑paste when updating files; avoid rewriting from scratch to be efficient.

---

### Phase 1 – Topic Identification & Initial Gathering
*Goal: compile a complete, validated list of all subjects / top‑level components that must be addressed – nothing missing.*

1. **Analyze** the task to identify exactly what information is required.
2. **Ask the user** for any missing details, clarifications, or ambiguous points — never assume.
3. **Search the web** as long as needed if external facts, current data, or contextual knowledge would improve accuracy.
4. **Inspect workspace files** if the task references or depends on them.
5. Collect all the **top lines**, the top subjects, the things we need to think about – like a list.  
   *(Example: for building a laptop the main lines could be RAM, screen, keyboard, etc.)*
6. **Validate** that the list is complete – every necessary topic is present, nothing is missing.
7. **Do not proceed** to deep diving until you have all topics and inputs ready.

---

### Phase 2 – Deep Dive per Topic
*Goal: for each topic from the list, get real depth – no fillers, only critical, practical, probability‑spanning detail.*

1. Take each topic from the validated list **one by one**.
2. For each topic, dedicate a full session (or more if needed) to **search and think deeply**.
   - Fill all gaps – get as deep as possible.
   - Focus on **real, deep info**: the critical aspects, the practical things, creating all possible probabilities, edge cases, dependencies, and optimal sequencing.
   - Use detailed **chain‑of‑thought reasoning**; take as much internal reasoning time as needed.
3. Continue until every topic has been exhaustively explored and all details are filled.

---

### Phase 3 – Consolidation, Conflict Resolution & Plan Creation
*Goal: review all collected knowledge, resolve conflicts, then craft a fully detailed, phased plan that anyone could follow.*

1. **Read** through all the filled‑out topic deep‑dives.
2. **Fix any conflicts** between topics and fill any remaining gaps – you should now have all the data needed.
3. With the complete data, begin **creating a detailed phased plan**.
   - Discuss and decide everything.
   - The plan must have **deep, granular detail** – so detailed that you could give it to another person and they could continue normally without further clarification.
4. The plan is now ready for execution.

---

### Phase 4 – Execution, Evaluation & Self‑Correction
*Goal: implement the plan flawlessly, step by step, correcting any imperfection immediately.*

1. **Implement** the plan one step at a time.
2. After each step, **evaluate** the result:
   - Is the output correct and complete?
   - Does it satisfy the original requirements and constraints?
   - Are there any errors, inefficiencies, or improvements?
3. If any flaw is found, **refine immediately** and re‑evaluate the step until it is perfect.
4. Continue this iterative cycle until all steps are flawlessly completed and the final answer is fully polished.
